# Блок 1. Программирование (1C-Bitrix)

Этот раздел содержит решения задач, специфичных для CMS 1С-Битрикс. Поскольку логика системы тесно завязана на структуру базы данных, ниже приведены инструкции по воссозданию окружения.

## Состав выполненных работ

* **Задание 1.3:** FAQ.
* **Задание 1.4:** Кастомный компонент.

---

## 1. Подготовка и развертывание

Чтобы код корректно отработал на вашей инсталляции Битрикс, выполните следующие шаги:

### Автоматическая настройка
В папке `/install` подготовлен XML-экспорт инфоблоков.
* Импортируйте файлы `iblock_1_3_export.xml` и `iblock_1_4_export.xml` через административную панель: *Контент -> Инфоблоки -> Импорт -> XML*.

### ⚠️ Важно после установки (Настройка ID)
При импорте Битрикс присвоит инфоблокам новые ID. Чтобы скрипты заработали, необходимо:
1. **Для задания 1.3:** открыть файл `local/php_interface/init.php` и заменить значение переменной `$faqIblockId = 5;` на актуальный ID инфоблока FAQ.
2. **Для задания 1.4:** передать новый ID инфоблока в параметры вызова компонента (`IBLOCK_ID`).

## Скриншоты
В папке `/screenshots` лежат скриншоты настроек инфоблоков и демонстрация работы компонента в браузере.

## 2. Описание реализации задач

### Задание 1.3: FAQ
**Логика:** Реализована через класс `FaqMailHandler` и обработчик события `OnAfterIBlockElementUpdate` в файле `init.php`.

**Особенности реализации:**
* **Защита от дублирования писем:** Отправка происходит только если поле `DETAIL_TEXT` (ответ) не пустое, email валиден (`check_email`), и кастомное свойство инфоблока `SENT_ANSWER` не заполнено. После успешной отправки свойству присваивается значение `Y` (через метод `SetPropertyValuesEx`, чтобы не вызывать зацикливание события обновления), блокируя повторные отправки при последующих редактированиях элемента.
* **Поддержка многосайтовости:** Скрипт динамически определяет `LID` (ID сайта) инфоблока через `\CIBlock::GetSite`. Это гарантирует использование корректного почтового шаблона.
* **API D7:** Для работы с почтой используется современный класс `\Bitrix\Main\Mail\Event`. Почтовое событие — `FAQ_ANSWER_MAIL`.

### Задание 1.4: Кастомный компонент
**Путь:** `/local/components/my_components/elements.list`

**Особенности реализации:**
* **ORM D7:** Для выборки данных использован класс `\Bitrix\Iblock\ElementTable` вместо старого `CIBlockElement::GetList`. Получение названия раздела реализовано без лишних запросов через связи сущностей (`IBLOCK_SECTION.NAME`).
* **Кэширование:** Выборка кэшируется с помощью `$this->startResultCache()`. Дополнительно внедрено тегированное кэширование (TaggedCache) по ID инфоблока (`iblock_id_X`), что гарантирует автоматический сброс кэша компонента при добавлении или изменении элементов контент-менеджером.
* **Архитектура:** Компонент инкапсулирован в класс `CustomIblockList` (наследует `\CBitrixComponent`). Время кэширования и ID инфоблока вынесены в настройки компонента (`.parameters.php`).

**Пример вызова:**
```php
$APPLICATION->IncludeComponent(
    "my_components:elements.list",
    ".default",
    [
        "IBLOCK_ID" => "5", // Заменить на актуальный ID инфоблока
        "CACHE_TIME" => "3600",
        "CACHE_TYPE" => "A"
    ]
);
